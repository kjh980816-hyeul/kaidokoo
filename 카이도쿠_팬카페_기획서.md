# 유령선장 카이도쿠 공식 팬카페 — 기획서

> 천문 항해(Celestial Navigation) × 유령선 컨셉 · 골드 × 미드나잇 네이비
> 씨미(SeeME) 스트리머 **유령선장 카이도쿠** 팬 커뮤니티
> 문서 버전 v0.1 (초안)

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 유령선장 카이도쿠 공식 팬카페 (가칭) |
| 대상 | 씨미(SeeME) 스트리머 **카이도쿠**의 팬·시청자 |
| 목적 | 방송 외 시간에도 팬이 모여 글·아트·클립을 나누고, 라이브 알림·출석으로 재방문을 유도하는 상시 커뮤니티 |
| 핵심 컨셉 | 별바다를 항해하는 **유령선장**. 안개 낀 밤하늘을 나침반 삼아 떠도는 유령선의 무드 |
| 톤 | 골드 선화 × 미드나잇 네이비. 신비롭고 고급스럽게, 으스스함은 우아하게(공포물 아님) |
| 형태 | 공개 사이트 + 별도 **관리자 운영 화면**을 갖춘 풀스택 웹 서비스 |
| 기준 시안 | 기존 "씨미 선장 팬카페" 프론트엔드 시안의 디자인·모션 시스템을 계승 |

### 차별 포인트 — "유령선장" 해석
기존 천문 항해 시스템(나침반·키홀 엠블럼·유성·등대) 위에 유령선 요소를 더한다.

- 살짝 깔리는 **안개(fog) / 성운 글로우**, 별빛 사이를 떠도는 **도깨비불(will-o'-wisp)** 입자
- **유령선 실루엣**(가는 골드 선화)과 **랜턴/해골 키홀** 모티프
- 카피 톤: "정박", "출항", "표류", "좌표", "안개 너머" 등 유령선 항해 어휘
- 색·고급스러움은 유지하되, 채도 낮은 푸른 안개로 으스스함을 은은하게 표현

## 2. 사용자 역할 & 권한

| 역할 | 설명 | 주요 권한 |
|---|---|---|
| 비회원(Guest) | 로그인 안 한 방문자 | 공개 게시판 열람, 라이브 배너 확인 |
| 회원(Member) | 가입·로그인 사용자 | 글/댓글 작성, 팬아트 업로드, 출석, 좋아요, 신고, 마이페이지 |
| 관리자(Admin) | 운영진(1~N명) | 관리자 화면 전체. 게시판 구성, 게시물·댓글·회원 관리, 라이브/사이트 설정 |

> 권한 등급은 `GUEST / MEMBER / ADMIN` 3단계로 시작. 추후 `SUPER_ADMIN`(운영진 관리·설정 변경 전용) 분리 가능.

## 4. 기능 명세 — 공개 사이트

### 4.1 로딩 화면 (인트로)
- 풀스크린. 중앙 **키홀 엠블럼**(점선+실선 동심원 + 사각틀 + 키홀) 회전, 뒤로 나침반 역회전.
- 별하늘 + 유성 + **안개/도깨비불 입자**가 함께 깔림.
- "Now Loading…" 또는 한글 카피, 얇고 자간 넓은 폰트로 페이드.
- 최소 노출 시간(약 2.6초) 후 부드럽게 본문으로 전환.
- 세션당 1회만 노출(재방문 시 스킵 옵션), 하단 "다시 보기" 버튼.
- `prefers-reduced-motion` 시 모션 최소화 + 즉시 전환.

### 4.2 홈
- **히어로**: 엠블럼 로고 + "유령선장 카이도쿠", 슬로건(예: "안개 너머, 별을 좇는 유령선").
- **라이브 알림 배너**(아래 4.3).
- **게시판 카드 그리드**: 관리자가 구성한 게시판을 데이터 기반으로 렌더(이름·아이콘·설명·글 수·최신 시각·노출/순서 모두 관리자 제어). 갤러리형은 별도 태그.
- **출석체크 위젯**(아래 4.6).
- 배경 별하늘·유성·안개는 전 페이지 공통, 성능 가벼운 수준 유지.

### 4.3 라이브 알림 배너
- 상태: **ON AIR**(방송 중) / **OFF AIR**(정박 중).
- ON: 방송 제목·시청자 수(또는 hint)·"방송 보기"(씨미 채널 링크) CTA.
- OFF: "정박 중" 문구 + "알림 받기" CTA.
- 갱신 방식 2안 (3.x 미해결 결정 참고):
  - (A) 씨미 라이브 상태 API/공개 페이지 연동 → 자동 갱신
  - (B) 미지원 시 **관리자 수동 토글** (관리자 화면에서 ON/OFF·제목·링크 입력)
- 등대(beacon) 모티프로 ON일 때 골드 신호 점등 애니메이션.

### 4.4 게시판 — 목록 / 상세 / 작성
- **목록**: 카드 또는 리스트형, 번호(№)·제목·작성자·작성시각·조회·좋아요·댓글수. 페이지네이션 또는 무한스크롤. 상단 고정(공지) 지원. 정렬(최신/인기), 검색.
- **상세**: 본문(에디터 렌더), 첨부 이미지, 좋아요, 신고, 댓글·대댓글, 이전/다음 글. 작성자/관리자만 수정·삭제.
- **작성/수정**: 제목·본문 에디터(이미지 업로드 포함), 게시판 선택(권한 내), 임시저장(선택). 비회원은 작성 불가(로그인 유도).
- 게시판은 **고정 아님** — 관리자가 추가/삭제/이름변경/순서/노출 제어. 코드에 게시판명 하드코딩 금지.

### 4.5 갤러리형 게시판 (팬아트)
- 어두운 배경에 작품이 별처럼 떠 있는 그리드, 호버 시 골드 테두리 점등.
- 라이트박스 상세(작품·작성자·좋아요·댓글). 이미지 최적화(썸네일/원본 분리).

### 4.6 출석체크
- 이번 달 날짜 수만큼 **별 도장**이 깔리고, 출석한 날 별이 점등.
- "오늘의 별 점등하기" → 완료 시 "오늘 항해 완료 ✦"로 비활성.
- **이번 달 출석 일수 + 연속 출석(streak)** 표시.
- 저장: 시안은 localStorage였으나 **실서비스는 서버(DB) 저장**으로 전환(회원 계정 귀속). 연속 출석 보상·뱃지 확장 가능.

### 4.7 회원
- **가입/로그인**: 씨미 연동 우선 검토, 미지원 시 소셜(네이버 등) 또는 자체 가입.
- **마이페이지**: 내 글·댓글·출석 현황·좋아요한 글·회원정보 수정·탈퇴.

### 4.8 통합 검색
- 게시판/제목/본문/작성자 대상 검색, 게시판별 필터.

## 5. 기능 명세 — 관리자 화면

> 공개 사이트와 동일한 골드×네이비 톤이되, **차분하고 정보 밀도 높은 운영 UI**.

### 5.1 대시보드
- 오늘 가입/방문/게시글/댓글/출석 요약, 신고 대기 건수, 라이브 현재 상태, 최근 활동 로그.

### 5.2 게시판 관리 (핵심)
- 게시판 **추가/수정/삭제**.
- 필드: 코드, 한글명, 영문명, 아이콘(선화 세트에서 선택), 설명, 정렬 순서(드래그), 노출 여부 토글, 타입(일반/갤러리), 글쓰기 권한.
- 변경 즉시 공개 사이트 카드 그리드에 반영.

### 5.3 게시물 관리
- 전체/게시판별 게시물 목록, 검색·필터.
- 상단 고정, 숨김/복구, 삭제(소프트 딜리트), 게시판 이동.

### 5.4 댓글 관리
- 최근 댓글·신고 댓글, 숨김/삭제.

### 5.5 신고 관리
- 게시물·댓글 신고 큐, 사유·신고자·대상 확인, 처리(반려/숨김/삭제/회원 제재) + 처리 로그.

### 5.6 회원 관리
- 회원 목록·검색, 상태(정상/정지/탈퇴), 권한 부여(관리자 지정), 활동 이력, 제재(정지·차단).

### 5.7 라이브 배너 관리
- ON/OFF 토글, 방송 제목·시청자 hint·채널 링크 입력.
- 씨미 자동 연동 사용 시: 연동 상태 표시 + 수동 오버라이드 옵션.

### 5.8 출석·이벤트 관리
- 출석 현황 통계, 연속 출석 보상/뱃지 설정(확장), 이벤트성 출석 보너스.

### 5.9 사이트 설정
- 테마 토큰(색/카피), 히어로 슬로건, 로딩 카피, 외부 링크(씨미 채널·SNS), 정책 문서, 운영진 표시.

## 6. 데이터 모델 (초안)

> 표기는 논리 모델. 실제 컬럼·인덱스·제약은 구현 단계에서 확정.

| 테이블 | 핵심 컬럼 | 비고 |
|---|---|---|
| `member` | id, login_type, external_id, nickname, role, status, created_at | role: GUEST/MEMBER/ADMIN |
| `board` | id, code, name_kr, name_en, icon, description, sort_order, type, is_visible, write_role, created_at | type: GENERAL/GALLERY |
| `post` | id, board_id(FK), author_id(FK), title, content, view_count, like_count, status, is_pinned, created_at, updated_at | status: PUBLISHED/HIDDEN/DELETED |
| `post_image` | id, post_id(FK), url, thumb_url, sort_order | 갤러리·첨부 |
| `comment` | id, post_id(FK), author_id(FK), parent_id, content, status, created_at | 대댓글: parent_id |
| `post_like` | member_id(FK), post_id(FK), created_at | (회원,글) 유니크 |
| `attendance` | id, member_id(FK), attend_date, created_at | (회원,날짜) 유니크 |
| `live_status` | id, is_live, title, viewer_hint, stream_url, updated_by, updated_at | 단일 행 또는 이력 |
| `report` | id, target_type, target_id, reporter_id(FK), reason, status, handled_by, created_at | target_type: POST/COMMENT |
| `site_setting` | key, value | 테마·카피·링크 (key-value) |
| `admin_log` | id, admin_id(FK), action, target, detail, created_at | 운영 감사 로그 |

**주요 관계**: member 1—N post / comment / attendance / report. board 1—N post. post 1—N comment / post_image / post_like.

## 9. 비기능 요구사항
- **성능**: 캔버스 별하늘·안개는 DPR 제한·입자 수 조절로 모바일 부하 관리.
- **접근성**: `prefers-reduced-motion`, 키보드 포커스, 대비 확보(어두운 배경 위 가독성).
- **SEO/공유**: 메타·OG 이미지(엠블럼 키홀), 게시판/글 공유 카드.
- **보안**: 인증·권한 검증(서버측), 신고/제재, 업로드 검증, 관리자 감사 로그.
- **운영**: 관리자 작업 로그(`admin_log`)로 추적 가능.

## 11. 미해결 결정사항 (확정 필요)

1. **씨미 로그인/라이브 연동** — 씨미가 OAuth·라이브 상태 API/공개 페이지를 제공하는지? 미지원이면 인증은 소셜·자체, 라이브는 관리자 수동 토글로 확정.
2. **서비스명·도메인** — "카이도쿠 팬카페"의 정식 명칭·URL.
3. **유령선 톤 강도** — 안개·으스스함을 어느 수위까지(우아한 신비 vs. 좀 더 다크) 갈지.

5. **관리자 인원·권한 분리** — ADMIN 단일인지, SUPER_ADMIN 분리할지.
