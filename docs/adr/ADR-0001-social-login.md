# ADR-0001 · 로그인 방식: 씨미 OAuth 단독 → 네이버/카카오/구글 소셜 3종

- 상태: 확정 (2026-06-05)
- 맥락: 핸드오프 CLAUDE.md §3.1 원안은 "로그인 = 씨미(CIME) OAuth 2.0 단독, 씨미를 IdP로 사용"이었다.

## 결정
로그인은 **네이버 + 카카오 + 구글 소셜 3종 OAuth 2.0**으로 한다. 씨미는 **로그인에서 제외**하고 **라이브 배너 전용**으로만 쓴다.

- 자체 가입 폼은 없다(소셜만).
- `member` 스키마는 멀티프로바이더: `provider`(NAVER/KAKAO/GOOGLE) + `provider_user_id`. (핸드오프의 `seeme_user_id`를 대체)
- 소셜 인증 성공 후 우리 서비스가 자체 세션 토큰을 발급한다(기존 설계 유지).

## 근거
- 팬 진입 장벽: 국내 사용자는 네이버/카카오 계정 보급률이 높아 씨미 단독보다 가입 마찰이 작다.
- 씨미 OpenAPI는 베타라 로그인 가용성·안정성 리스크가 있다. 씨미는 라이브 상태 확인이라는 약한 의존으로 격리.
- "씨미 계정 연동"은 후순위 확장 여지로 남긴다(필요 시 별도 연동 버튼).

## 영향
- 고추밭(gochubat)의 네이버 OAuth 구현을 이식하고 카카오/구글을 추가.
- 씨미 연동 규칙(`rules/project/10-seeme-integration.md`)에서 OAuth 로그인 섹션은 **라이브 배너 한정**으로 해석한다(로그인 무관).
- 소셜 앱 등록은 준하님이 직접 수행, 키는 서버 환경변수로만 보관.
